<details>
<summary><strong>SQL ‚úÖ ‚Äî Strongest Area</strong></summary>

* `SELECT` `WHERE` `DISTINCT`
* `ORDER BY` `LIMIT` `OFFSET`
* `GROUP BY` `HAVING`
* Aggregate Functions ‚Äî `COUNT` `SUM` `AVG` `MIN` `MAX` ¬∑ `ROUND`
* `CASE WHEN` ‚Äî simple, multi-column, with GROUP BY, with JOINs, pivot-style conditional aggregation
* JOINs ‚Äî INNER, LEFT, RIGHT, SELF, Multi-table, Anti-Join
* JOIN + GROUP BY ‚Äî city-wise revenue, category-wise quantity, avg order amount
* Subqueries ‚Äî Scalar, `IN`/`NOT IN`, Derived Tables, Correlated, Nested (2nd highest salary, nested FROM in HAVING)
* Window Functions ‚Äî `ROW_NUMBER` `RANK` `DENSE_RANK` `LAG` `LEAD` `FIRST_VALUE` `LAST_VALUE` `SUM() OVER` `AVG() OVER` `PERCENT_RANK()` ¬∑ `PARTITION BY` ¬∑ `ROWS BETWEEN` frame clause ¬∑ `WINDOW` alias
* CTEs ‚Äî basic (single), chained (multi-CTE), CTE + Window Function (TOP-N filter), recursive CTE (`WITH RECURSIVE` ‚Äî hierarchy tree, level tracking, path building) ¬∑ `IFNULL` ¬∑ `CROSS JOIN` for scalar CTE values
* String Functions ‚Äî `UPPER` `LOWER` `LENGTH` `CHAR_LENGTH` `SUBSTRING` `INSTR` `REPLACE` `CONCAT` `TRIM` `LIKE` `LPAD` `RPAD` `LEFT` `RIGHT` `GROUP_CONCAT`
* `BETWEEN` / `NOT BETWEEN`
* Date Functions ‚Äî `MONTH()` `YEAR()` `MONTHNAME()`
